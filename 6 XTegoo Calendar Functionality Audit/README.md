Проект № 6 - Calendar & Scheduling Features Testing

Практический проект по ручному тестированию функционала календаря и планирования встреч в мобильном мессенджере XTegoo.

В рамках проекта проведено тестирование ключевых сценариев, связанных с созданием, редактированием и совместным использованием событий календаря, а также проверкой уведомлений и интеграции с бизнес‑аккаунтом.
Цель проекта

Проверить корректность работы календарного функционала приложения, убедиться в удобстве планирования встреч и обмена расписанием, а также выявить дефекты, влияющие на пользовательский опыт при работе с календарём.
Объект тестирования

Мобильное приложение XTegoo Messenger

    Платформа: Android

    Тип тестирования: Manual QA

Области тестирования
1. Создание и редактирование событий

Проверено:

    доступность интерфейса создания события;

    корректность заполнения полей (название, дата, время, описание);

    возможность установки напоминаний;

    сохранение и редактирование созданного события;

    удаление событий из календаря.

2. Совместное использование и обмен расписанием

Проверено:

    возможность поделиться событием с другим пользователем;

    отображение приглашений на события у получателя;

    принятие/отклонение приглашений;

    обновление статуса события при изменении организатором;

    настройка уровней доступа (просмотр/редактирование).

3. Интеграция с бизнес‑аккаунтом и уведомления

Проверено:

    привязка событий к бизнес‑контактам;

    отображение информации о бизнес‑участниках в событиях;

    корректная работа push‑уведомлений о предстоящих встречах;

    уведомления об изменениях в расписании;

    синхронизация календаря с другими устройствами.

4. Визуальное отображение и навигация

Проверено:

    корректное отображение календаря в разных режимах (день, неделя, месяц);

    визуальная индикация событий и напоминаний;

    плавность прокрутки и переходов между режимами;

    отсутствие визуальных дефектов и ошибок отображения.

Особенности тестирования

Тестирование проводилось с учётом:

    созданного бизнес‑аккаунта с заполненным профилем;

    настроенных параметров приватности календаря;

    нескольких тестовых пользователей для проверки обмена событиями;

    разных сценариев планирования (личные и рабочие встречи);

    проверки на разных разрешениях экрана.

Результаты тестирования

В ходе тестирования выявлены следующие дефекты:

    уведомления о новых событиях приходят с задержкой;

    в режиме «неделя» некоторые события не отображаются;

    при отклонении приглашения организатор не получает уведомление;

    настройки приватности не применяются к новым событиям автоматически;

    на маленьких экранах текст в событиях обрезается.

Структура проекта

    test-cases — тест‑кейсы для всех сценариев работы с календарём;

    bug-reports — подробные баг‑репорты с шагами воспроизведения и скриншотами;

    screenshots — визуальные доказательства дефектов и корректной работы;

    test-results — сводный отчёт по результатам тестирования.

Вывод

Календарный функционал приложения предоставляет базовые возможности планирования и обмена событиями, но требует доработки в части уведомлений и отображения данных.

Особое внимание требуется уделить:

    своевременности push‑уведомлений;

    корректному отображению событий в разных режимах календаря;

    улучшению механизма обратной связи при работе с приглашениями;

    оптимизации интерфейса для устройств с маленькими экранами;

    автоматизации применения настроек приватности.
