# 7 - Проверка отображения аватарки профиля спустя время

Приоритет: Высокий (High)
Тип: Функциональный / Стабильность данных
Предусловие:

    Аватарка для пользователя добавлена день назад.

    На момент добавления аватарка корректно отображается в профиле.

Шаги выполнения:

    На следующий день после добавления аватарки открыть приложение XTegoo Messenger.

    Перейти в «Настройки».

Ожидаемый результат: аватарка пользователя отображается в настройках профиля без изменений.
Фактический результат: спустя 2 дня добавленная ранее аватарка отсутствует (отображается стандартный аватар или пустое поле).
Статус: Failed
Окружение: iOS 16.5, XTegoo Messenger v2.3.1
Воспроизводимость: 80 % (в 4 из 5 попыток аватарка пропадала на 2‑й день)

Связанный баг‑репорт: BUG‑08
Вложения: screenshots/bug_08_avatar_missing.png

Рекомендации по исправлению:

    проверить механизм хранения и кэширования аватарки;

    убедиться, что сервер не сбрасывает изображение по истечении срока;

    проверить синхронизацию между клиентом и сервером;

    проанализировать логи загрузки изображения при открытии настроек.

Дополнение к баг‑репортам
BUG‑08: Пропажание аватарки профиля спустя 24–48 часов

Заголовок: Аватарка профиля пропадает спустя 1–2 дня после загрузки
Приоритет: Высокий (High)
Серьёзность: Высокая (High) — влияет на идентификацию пользователя и восприятие профиля

Шаги воспроизведения:

    Добавить аватарку в профиль.

    Убедиться, что она отображается (день 1).

    На следующий день (день 2) открыть настройки приложения.

Ожидаемый результат: аватарка сохраняется и отображается в настройках.
Фактический результат: аватарка отсутствует, отображается стандартный силуэт или пустое поле.

Окружение:

    iOS 16.5, XTegoo Messenger v2.3.1;

    стабильное интернет‑соединение.

Воспроизводимость: 80 %
Вложения: screenshots/bug_08_avatar_missing.png

Описание проблемы:
Аватарка, успешно загруженная и отображаемая в день добавления, исчезает из интерфейса на 2–3‑й день. Проблема наблюдается как при повторном открытии приложения, так и при перезагрузке устройства.

Возможные причины:

    истечение срока кэширования изображения на клиенте;

    ошибка синхронизации с сервером (потеря ссылки на файл);

    автоматическое удаление старых аватарок на бэкенде;

    проблема с правами доступа к файлу изображения.

Рекомендации по исправлению:

    Проверить политику кэширования медиафайлов в приложении.

    Проанализировать запросы к серверу при загрузке аватарки.

    Убедиться, что URL изображения сохраняется и повторно загружается корректно.

    Проверить логи сервера на наличие ошибок удаления файлов.
